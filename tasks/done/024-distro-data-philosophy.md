# Task 024 – Distro Data Philosophy & Rules

## Objective
Formally define how Linux distributions are represented in the system.

This task establishes that distros are **constraint-bearing entities**, not recommendations.

---

## Scope
- docs/DISTRO_DATA_PHILOSOPHY.md

---

## Requirements
The document must clearly state:
- Distros are facts, not opinions
- No distro can be labeled “best”, “beginner”, or “recommended”
- All attributes must be verifiable and deterministic
- Trade-offs must be explicit

---

## Definition of Done
- Document exists
- Language is non-opinionated
- Can be referenced by agents when adding distro data
